The `Try` type wraps synchronous functions that might throw exceptions. It captures the result or the error without using try-catch blocks.

## The Exception Problem

In JavaScript/TypeScript, many functions can throw exceptions:

```typescript
const parseJSON = (json: string): object => {
  return JSON.parse(json); // Can throw on invalid JSON!
};

const getConfig = (): object => {
  return JSON.parse(fs.readFileSync("config.json", "utf-8"));
  // Can throw if file doesn't exist!
};
```

When using these functions, you're forced to use try-catch:

```typescript
try {
  const data = parseJSON(userInput);
  // use data...
} catch (e) {
  console.error("Failed to parse:", e);
  // But what if we need to handle this in the type system?
}
```

Problems with this approach:
1. Exceptions can escape if you forget a catch
2. The return type doesn't indicate potential failure
3. Error handling becomes deeply nested

## Enter Try: Capturing Exceptions

Try wraps a function and captures any exception:

```typescript
import { attempt, isOk, isErr } from "@deessejs/core";

const parseJSON = (json: string): Try<object> =>
  attempt(() => JSON.parse(json));

const result = parseJSON('{"valid": true}');

if (isOk(result)) {
  console.log(result.value); // { valid: true }
}

if (isErr(result)) {
  console.error(result.error); // SyntaxError: Unexpected token...
}
```

Now the **potential for failure is explicit in the type**.

## Imperative vs Functional

Here's how you'd traditionally handle file operations:

```typescript
import * as fs from "fs";

function readConfig(): object {
  try {
    const content = fs.readFileSync("config.json", "utf-8");
    return JSON.parse(content);
  } catch (e) {
    throw new Error(`Failed to read config: ${e}`);
  }
}

try {
  const config = readConfig();
} catch (e) {
  // Handle error
}
```

With Try, you get explicit error handling:

```typescript
import { attempt, Try } from "@deessejs/core";
import * as fs from "fs";

const readConfig = (): Try<object, Error> =>
  attempt(() => JSON.parse(fs.readFileSync("config.json", "utf-8")));

// The error type is now explicit in the return type!
const result = readConfig()
  .map(cfg => processConfig(cfg))
  .getOrElse({ defaults: true });
```

## Overview

```typescript
type Try<T, E = Error> = TrySuccess<T> | TryFailure<E>;
```

- **`TrySuccess<T>`** - Represents a successful execution with value `T`
- **`TryFailure<E>`** - Represents a failure with error `E`

## Creating a Try

### attempt - Wrap a Function

Capture success:

```typescript
import { attempt, isOk, isErr } from "@deessejs/core";

const result = attempt(() => {
  return 42;
});

console.log(isOk(result)); // true
console.log(result.value); // 42
```

Capture failure:

```typescript
import { attempt, isErr } from "@deessejs/core";

const result = attempt(() => {
  throw new Error("Something went wrong");
});

console.log(isErr(result)); // true
console.log(result.error.message); // "Something went wrong"
```

### attemptAsync - Wrap Async Functions

For async functions that might throw:

```typescript
import { attemptAsync } from "@deessejs/core";

const fetchData = async () => {
  const response = await fetch("/api/data");
  return response.json();
};

const result = await attemptAsync(fetchData);
```

## Transforming Try

### map - Transform the Value

When you want to transform the contained value:

```typescript
import { attempt, map } from "@deessejs/core";

const result = attempt(() => 10)
  .map(x => x * 2)
  .map(x => x + 1);

console.log(result.value); // 21
```

If it threw, transformations are skipped:

```typescript
import { attempt, map } from "@deessejs/core";

const result = attempt(() => { throw new Error("fail"); })
  .map(x => x * 2);
// Error is preserved
```

### flatMap - Chain Operations

When your transformation returns a Try:

```typescript
import { attempt, flatMap } from "@deessejs/core";

const parseNumber = (s: string): Try<number> =>
  attempt(() => parseInt(s, 10));

const result = attempt(() => "42")
  .flatMap(parseNumber);
```

## Handling Try

### getOrElse - Get Value or Default

```typescript
import { attempt, getOrElse } from "@deessejs/core";

attempt(() => 10).getOrElse(0); // 10
attempt(() => { throw new Error(); }).getOrElse(0); // 0
```

### getOrCompute - Compute Default Lazily

```typescript
import { attempt, getOrCompute } from "@deessejs/core";

const result = attempt(() => { throw new Error(); })
  .getOrCompute(() => 42);
```

### match - Pattern Matching

```typescript
import { attempt, match } from "@deessejs/core";

const result = attempt(() => 42);
const message = match(
  result,
  (value) => `Success: ${value}`,
  (error) => `Failed: ${error.message}`
);
```

### tap - Side Effects

```typescript
import { attempt, tap } from "@deessejs/core";

attempt(() => 42)
  .tap(value => console.log(value)); // Logs: 42
```

### tapErr - Side Effects on Error

```typescript
import { attempt, tapErr } from "@deessejs/core";

attempt(() => { throw new Error("fail"); })
  .tapErr(error => console.error(error));
```

## Type Guards

### isOk / isErr

```typescript
import { attempt, isOk, isErr } from "@deessejs/core";

const result = attempt(() => 42);

if (isOk(result)) {
  console.log(result.value); // 42
}

if (isErr(result)) {
  console.log(result.error);
}
```

## When to Use Try

Use Try when:

- Wrapping synchronous functions that might throw
- Working with APIs that use exceptions for error handling
- Converting legacy code to type-safe code
