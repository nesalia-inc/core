This is a minimal API reference. For detailed explanations, see the Guides and Concepts sections.

## Creating

| Function | Description |
|----------|-------------|
| `ok(value)` | Create successful Result |
| `err(error)` | Create failed Result |
| `some(value)` | Create present Maybe |
| `someUnit()` | Create Some with undefined value |
| `none()` | Create absent Maybe |
| `fromNullable(value)` | Convert null/undefined to Maybe |
| `fromUndefinedable(value)` | Convert undefined to None |
| `okAsync(value)` | Create successful AsyncResult |
| `errAsync(error)` | Create failed AsyncResult |
| `fromPromise(promise)` | Wrap Promise to AsyncResult |
| `attempt(fn)` | Wrap sync function that might throw |
| `attemptAsync(fn)` | Wrap async function |
| `error(options)` | Create Error type builder (Python-like) |
| `exceptionGroup(errors)` | Create ErrorGroup with multiple errors |
| `raise(error)` | Functional throw for use in transformations |

## Error System

| Function | Description |
|----------|-------------|
| `error(options)` | Create structured Error type with name, args, notes, cause |
| `exceptionGroup(errors)` | Group multiple errors together |
| `raise(err)` | Convert Err for use in transformations |
| `isError(value)` | Type guard for Error |
| `isErrorGroup(value)` | Type guard for ErrorGroup |
| `isErrWithError(result)` | Check if Result contains Error |
| `isErrTryWithError(t)` | Check if Try contains Error |
| `getErrorMessage(error)` | Get formatted error message |
| `flattenErrorGroup(group)` | Flatten nested ErrorGroups |
| `filterErrorsByName(group, name)` | Filter errors by name |

## Transforming

| Function | Description |
|----------|-------------|
| `map(result, fn)` | Transform the value |
| `flatMap(result, fn)` | Chain operations that return Result |
| `mapErr(result, fn)` | Transform the error |

## Extracting

| Function | Description |
|----------|-------------|
| `getOrElse(result, default)` | Get value or default |
| `getOrCompute(result, fn)` | Compute default lazily |
| `match(result, ok, err)` | Pattern matching |
| `tap(result, fn)` | Side effects without changing value |
| `tapErr(result, fn)` | Side effects on error |

## Type Guards

| Function | Description |
|----------|-------------|
| `isOk(result)` | Check if Ok |
| `isErr(result)` | Check if Err |
| `isSome(maybe)` | Check if Some |
| `isNone(maybe)` | Check if None |
| `isError(value)` | Check if Error |
| `isErrorGroup(value)` | Check if ErrorGroup |
| `isErrWithError(result)` | Check if Result contains Error |
| `isErrTryWithError(t)` | Check if Try contains Error |

## Converting

| Function | Description |
|----------|-------------|
| `toResult(maybe, onNone)` | Maybe → Result |
| `toMaybeFromResult(result)` | Result → Maybe |

## Async Utilities

| Function | Description |
|----------|-------------|
| `all(...results)` | Run multiple AsyncResults in parallel |
| `race(...results)` | First AsyncResult to resolve |
| `traverse(items, fn)` | Map array to AsyncResults |
| `retryAsync(fn, options)` | Retry failed async operations |
| `retry(fn, options)` | Retry sync operations |
| `sleep(ms)` | Async delay |
| `withTimeout(ms, fn)` | Add timeout to async operation |
| `sleepWithSignal(signal)` | Sleep that responds to AbortSignal |
| `exponentialBackoff(attempt, delay)` | Exponential backoff strategy |
| `linearBackoff(attempt, delay)` | Linear backoff strategy |
| `constantBackoff(attempt, delay)` | Constant backoff strategy |

## Options & Config

### RetryOptions

```typescript
interface RetryOptions {
  attempts?: number;        // Number of attempts (default: 3)
  delay?: number;          // Initial delay in ms (default: 1000)
  backoff?: "exponential" | "linear" | "constant" | ((attempt: number, delay: number) => number);
  predicate?: (error: Error) => boolean;
  onRetry?: (error: Error, attempt: number) => void;
  jitter?: boolean;
}
```

### ErrorOptions

```typescript
type ErrorOptions<T> = {
  name: string;
  args: T;
  defaultDescription?: string;
};

type ZodErrorOptions<T> = {
  name: string;
  schema: ZodSchema<T>;
  defaultDescription?: string;
};
```

### Error Type

```typescript
type Error<T = unknown> = {
  readonly name: string;
  readonly args: T;
  readonly notes: readonly string[];
  readonly cause: Error | null;
};

type ErrorGroup = {
  readonly name: string;
  readonly exceptions: readonly Error[];
};
```

## Quick Lookup

Looking for something specific?

- **Validation** → Use `ok()` + `err()` + `Result`
- **Optional values** → Use `some()` + `none()` + `Maybe`
- **Parsing JSON** → Use `attempt()` + `Try`
- **Network calls** → Use `AsyncResult`
- **Rich errors** → Use `Result` with Error types
- **Advanced error handling** → Use `error()` + `exceptionGroup()` (Python-inspired)

See [Choosing a Type](/docs/concepts/choosing-type) for detailed guidance.
