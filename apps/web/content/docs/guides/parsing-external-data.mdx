You need to parse JSON from an API, read a config file, or convert string data to typed objects. These operations can fail in predictable ways.

## The Problem

Parsing external data is risky:

```typescript
interface User {
  id: number;
  name: string;
  email: string;
}

const parseUser = (json: string): User => {
  return JSON.parse(json); // Can throw!
};

// What if the JSON is invalid?
// What if required fields are missing?
// What if types are wrong?
```

Problems:
- `JSON.parse` can throw SyntaxError
- Missing fields cause runtime errors
- Type coercion happens silently

<Callout type="warning">
  The return type says `User`, but this function can definitely throw. TypeScript doesn't tell you.
</Callout>

## The Solution: Try for Sync, Result for Validation

### Use Try for Operations That Might Throw

```typescript
import { attempt, Try, isOk, isErr } from "@deessejs/core";

const parseJson = <T>(json: string): Try<T> => {
  return attempt(() => JSON.parse(json) as T);
};

const result = parseJson<User>('{"id": 1, "name": "John"}');

if (isOk(result)) {
  console.log(result.value.name); // "John"
}

if (isErr(result)) {
  console.error("Parse failed:", result.error.message);
}
```

### Use Result for Validation

```typescript
import { ok, err, Result } from "@deessejs/core";

interface User {
  id: number;
  name: string;
  email: string;
}

const validateUser = (data: unknown): Result<User, string> => {
  if (!data || typeof data !== "object") {
    return err("Data must be an object");
  }

  const obj = data as Record<string, unknown>;

  if (typeof obj.id !== "number") {
    return err("id must be a number");
  }

  if (typeof obj.name !== "string" || obj.name.length === 0) {
    return err("name must be a non-empty string");
  }

  if (typeof obj.email !== "string" || !obj.email.includes("@")) {
    return err("invalid email");
  }

  return ok({
    id: obj.id,
    name: obj.name,
    email: obj.email,
  });
};
```

## Parsing Configuration Files

A common pattern: read a config file, parse it, validate it:

```typescript
import { attempt, ok, err, Result, map, getOrElse } from "@deessejs/core";
import * as fs from "fs";

interface Config {
  apiUrl: string;
  timeout: number;
  retries: number;
}

const parseConfig = (json: string): Result<Config, string> => {
  return attempt(() => JSON.parse(json))
    .mapErr(e => `Parse error: ${e.message}`)
    .andThen(data => validateConfig(data));
};

const validateConfig = (data: unknown): Result<Config, string> => {
  if (!data || typeof data !== "object") {
    return err("Config must be an object");
  }

  const obj = data as Record<string, unknown>;

  const apiUrl = obj.apiUrl;
  if (typeof apiUrl !== "string") {
    return err("apiUrl must be a string");
  }

  const timeout = obj.timeout;
  if (typeof timeout !== "number" || timeout < 0) {
    return err("timeout must be a positive number");
  }

  const retries = obj.retries ?? 3;

  return ok({ apiUrl, timeout, retries });
};

const loadConfig = (): Config => {
  const content = fs.readFileSync("config.json", "utf-8");
  return parseConfig(content)
    .getOrElse({ apiUrl: "http://localhost", timeout: 5000, retries: 3 });
};
```

## Parsing Environment Variables

Environment variables always come as strings:

```typescript
import { fromNullable, ok, err, Result, map, getOrElse } from "@deessejs/core";

interface EnvConfig {
  apiUrl: string;
  port: number;
  debug: boolean;
}

const parseEnv = (env: Record<string, string | undefined>): Result<EnvConfig, string[]> => {
  const errors: string[] = [];

  const apiUrl = fromNullable(env.API_URL)
    .mapErr(() => "API_URL is required")
    .mapErr(e => { errors.push(e); return e; });

  const port = fromNullable(env.PORT)
    .map(Number)
    .mapErr(() => "PORT must be a number")
    .mapErr(e => { errors.push(e); return e; });

  const debug = fromNullable(env.DEBUG)
    .map(v => v === "true")
    .getOrElse(false);

  if (errors.length > 0) {
    return err(errors);
  }

  return ok({
    apiUrl: apiUrl.getOrElse(""),
    port: port.getOrElse(3000),
    debug,
  });
};
```

## Recap

- Use `Try` to wrap operations that might throw (JSON.parse, file reads)
- Use `Result` to validate parsed data
- Chain `attempt().mapErr().andThen()` for parse + validate workflows
