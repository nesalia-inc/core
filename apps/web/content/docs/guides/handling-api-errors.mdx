Network requests fail. Servers go down. Timeouts happen. How do you handle this gracefully?

## The Problem

Traditional async error handling is messy:

```typescript
async function fetchUser(id: string): Promise<User> {
  const response = await fetch(`/api/users/${id}`);

  if (!response.ok) {
    throw new Error(`HTTP ${response.status}`);
  }

  return response.json();
}

// Caller must try-catch
try {
  const user = await fetchUser("123");
} catch (e) {
  // What kind of error is this?
  // Network error? HTTP error? Parse error?
}
```

Problems:
- All errors look the same
- No type safety on what can go wrong
- Easy to forget to handle errors

## The Solution: AsyncResult

Use `AsyncResult` to make async errors explicit:

```typescript
import { okAsync, errAsync, AsyncResult } from "@deessejs/core";

interface ApiError {
  code: "NETWORK_ERROR" | "HTTP_ERROR" | "PARSE_ERROR";
  status?: number;
  message: string;
}

async function fetchUser(id: string): AsyncResult<User, ApiError> {
  try {
    const response = await fetch(`/api/users/${id}`);

    if (!response.ok) {
      return errAsync({
        code: "HTTP_ERROR",
        status: response.status,
        message: `HTTP ${response.status}`
      });
    }

    const data = await response.json();
    return okAsync(data as User);

  } catch (e) {
    return errAsync({
      code: "NETWORK_ERROR",
      message: e instanceof Error ? e.message : "Unknown error"
    });
  }
}
```

Now callers know exactly what can go wrong:

```typescript
const result = await fetchUser("123");

result.match(
  (user) => {
    console.log("Got user:", user.name);
  },
  (error) => {
    switch (error.code) {
      case "NETWORK_ERROR":
        showToast("Network error. Check your connection.");
        break;
      case "HTTP_ERROR":
        showToast(`Error: ${error.message}`);
        break;
    }
  }
);
```

## Handling Different Error Types

Different errors need different handling:

```typescript
const handleResult = async (result: AsyncResult<User, ApiError>) => {
  // Use getOrElse for simple defaults
  const user = await result.getOrElse({
    id: 0,
    name: "Guest",
    email: ""
  });

  // Use match for different handling
  await result.match(
    (user) => {
      analytics.track("user_loaded", { userId: user.id });
      renderUser(user);
    },
    (error) => {
      if (error.code === "NETWORK_ERROR") {
        showOfflineIndicator();
      } else if (error.status === 404) {
        show404Page();
      } else {
        showErrorToast(error.message);
      }
    }
  );
};
```

## Adding Retry Logic

Network errors are often transient. Add retry:

```typescript
import { okAsync, errAsync, retryAsync } from "@deessejs/core";

const fetchWithRetry = async <T>(
  url: string
): Promise<T> => {
  return retryAsync(
    async () => {
      const response = await fetch(url);

      if (!response.ok) {
        throw new Error(`HTTP ${response.status}`);
      }

      return response.json() as Promise<T>;
    },
    {
      attempts: 3,
      delay: 1000,
      backoff: "exponential",
      // Only retry on network errors
      predicate: (error) => error.message.includes("fetch")
    }
  );
};
```

## Chaining API Calls

Need to call multiple APIs in sequence:

```typescript
import { okAsync, errAsync, flatMap } from "@deessejs/core";

const fetchUserAndPosts = async (userId: string) => {
  const userResult = await fetchUser(userId);

  // If user fetch fails, propagate the error
  const postsResult = await flatMap(
    userResult,
    async (user) => {
      const posts = await fetchPosts(user.id);
      return okAsync({ user, posts });
    }
  );

  return postsResult;
};

// Usage
const result = await fetchUserAndPosts("123");

result.match(
  ({ user, posts }) => {
    console.log(`${user.name} has ${posts.length} posts`);
  },
  (error) => {
    console.error("Failed:", error.message);
  }
);
```

## Recap

- Use `AsyncResult<T, Error>` for async operations
- Define explicit error types for different failure modes
- Use `retryAsync` for transient network errors
- Chain operations with `flatMap`
