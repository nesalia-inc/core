The `Outcome` type provides rich error handling with three variants: `Success` for successful operations, `Cause` for expected/business errors, and `Exception` for unexpected system errors.

## Why Outcome?

Sometimes a simple success/failure isn't enough. You need to distinguish between:

- **Business errors** - Expected failures your code should handle gracefully (validation errors, not found)
- **System errors** - Unexpected failures you probably can't recover from (network errors, bugs)

This distinction helps you decide how to handle each case:

```typescript
// With Result, both look the same
const result: Result<User, Error> = await fetchUser(id);

// With Outcome, you know the difference
const outcome: Outcome<User, ValidationError, SystemError> = await fetchUser(id);
```

<Callout>
  Outcome gives you three states: Success (happy path), Cause (expected errors you handle), and Exception (unexpected errors you might log and surface).
</Callout>

## Overview

```typescript
type Outcome<T, C = unknown, E = unknown> = Success<T> | Cause<C> | Exception<E>;
```

- **`Success<T>`** - Represents a successful operation with value `T`
- **`Cause<C>`** - Represents an expected error with cause data `C`
- **`Exception<E>`** - Represents an unexpected error with exception data `E`

## Creating an Outcome

### success - Create a Success

```typescript
import { success, isSuccess } from "@deessejs/core";

const outcome = success(42);
console.log(isSuccess(outcome)); // true
console.log(outcome.value); // 42
```

### successUnit - Create Unit Success

```typescript
import { successUnit } from "@deessejs/core";

const outcome = successUnit(); // Success<void>
```

### cause - Create a Cause (Business Error)

```typescript
import { cause, isCause } from "@deessejs/core";

const outcome = cause({
  name: "VALIDATION_ERROR",
  message: "Invalid email format",
  data: { field: "email" }
});

console.log(isCause(outcome)); // true
console.log(outcome.message); // "Invalid email format"
```

### causeUnit - Create Unit Cause

```typescript
import { causeUnit } from "@deessejs/core";

const outcome = causeUnit();
// Cause with undefined data
```

### exception - Create an Exception (System Error)

```typescript
import { exception, isException } from "@deessejs/core";

const outcome = exception({
  name: "NETWORK_ERROR",
  message: "Connection refused"
});

console.log(isException(outcome)); // true
console.log(outcome.name); // "NETWORK_ERROR"
```

### exceptionWithStack - With Stack Trace

```typescript
import { exceptionWithStack } from "@deessejs/core";

const outcome = exceptionWithStack({
  name: "ERROR",
  message: "Failed"
});
// Automatically captures stack trace
```

### exceptionUnit - Create Unit Exception

```typescript
import { exceptionUnit } from "@deessejs/core";

const outcome = exceptionUnit();
```

## Type Guards

### isSuccess - Check for Success

```typescript
import { success, cause, isSuccess } from "@deessejs/core";

isSuccess(success(42)); // true
isSuccess(cause({ name: "E", message: "e" })); // false
```

### isCause - Check for Cause

```typescript
import { cause, isCause } from "@deessejs/core";

isCause(cause({ name: "E", message: "e" })); // true
```

### isException - Check for Exception

```typescript
import { exception, isException } from "@deessejs/core";

isException(exception({ name: "E", message: "e" })); // true
```

## When to Use Outcome

Use Outcome when:

- You need to distinguish between business errors and system errors
- You want different handling strategies for expected vs unexpected failures
- You need rich error context (name, message, data)

| Function | Description |
